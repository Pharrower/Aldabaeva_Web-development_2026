# Информационная система автоматизации волонтерского приюта «Доброе сердце»

Курсовой проект по дисциплине «Технологии программирования» / «Разработка веб-приложений».
**Автор:** Алдабаева Виктория
**Стек технологий:** Java 25, Spring Boot 3.x, Spring Data JPA, PostgreSQL 17, Thymeleaf, Bootstrap 5, Apache Tomcat 10.1.

---

## 🐾 Описание проекта

Проект представляет собой динамическое веб-приложение для координации деятельности приюта для бездомных животных «Доброе сердце». Система решает ключевые задачи учета ресурсов, автоматизации взаимодействия с потенциальными владельцами и прозрачного ведения отчетности волонтерской организации.

### Ключевой функционал:
* **Каталог питомцев:** Интерактивный просмотр животных, находящихся в приюте, с возможностью асинхронной фильтрации.
* **Подача заявок на усыновление (Adoption):** Двухконтурная система валидации анкет будущих хозяев (клиентский JavaScript + серверный Jakarta Bean Validation).
* **Мониторинг склада материального обеспечения:** Учет кормов, медикаментов и хозяйственных товаров с визуализацией заполненности склада через динамические компоненты (Progress Bars).
* **Асинхронное взаимодействие (AJAX):** Отправка форм обратной связи и подгрузка данных без полной перезагрузки страниц с помощью Fetch API.

## 🛠 Архитектурная структура (Паттерн PCMEF)

Проект спроектирован по каноническому промышленному паттерну **PCMEF** (Presentation, Controller, Model, Entity, Foundation):

* `presentation/` (`templates/`) — визуальный слой приложения (HTML5, Thymeleaf, Bootstrap 5, JS/Fetch API).
* `controllers/` — REST и Web-контроллеры Spring, обеспечивающие маршрутизацию и обработку HTTP-запросов (`GET`, `POST`).
* `models/` (`services/`) — слой бизнес-логики приложения, транзакционные сервисы управления сущностями.
* `entities/` — доменная модель данных и ORM-маппинг (`User`, `Animal`, `AdoptionRequest`, `Supply`).
* `foundation/` (`repositories/` / `config/`) — базовый инфраструктурный слой (репозитории Spring Data JPA, конфигурация пула соединений HikariCP).

## 🚀 Инструкция по локальному развертыванию

### 1. Требования к окружению
* **СУБД:** PostgreSQL 17+ (порт по умолчанию `5432`)
* **Java SDK:** Версия 25
* **Сборщик:** Maven 3.9+
* **Сервер приложений:** Apache Tomcat 10.1.x

### 2. Подготовка базы данных
1. Создайте в PostgreSQL пустую базу данных с именем `shelter_db`.
2. Выполните SQL-скрипт инициализации таблиц и тестовых данных из папки репозитория:
```bash
   psql -U postgres -d shelter_db -f db/init.sql
```

### 3. Конфигурация приложения
Перед сборкой убедитесь, что в файле src/main/resources/application.properties корректно указаны ваши доступы к БД:

Properties
spring.datasource.url=jdbc:postgresql://localhost:5432/shelter_db
spring.datasource.username=your_postgres_user
spring.datasource.password=your_postgres_password

### 4. Сборка и деплой на Apache Tomcat 10.1
Поскольку проект разработан для промышленной эксплуатации на внешнем сервере приложений Tomcat, процесс сборки выглядит следующим образом:

1. Сборка WAR-архива:
В корне проекта выполните команду Maven для очистки и сборки артефакта:

Bash
   mvn clean package
В каталоге target/ будет сгенерирован файл приложения (например, shelter.war).

2. Настройка Tomcat:
Перейдите в конфигурационную директорию вашего установленного Tomcat (apache-tomcat-10.1.x/conf/) и в файле server.xml настройте выделенный сетевой порт (в проекте используется порт 8083 во избежание конфликтов):

XML
   <Connector connectionTimeout="20000" maxParameterCount="1000" port="8083" protocol="HTTP/1.1" redirectPort="8443"/>

3. Деплой приложения:

Скопируйте скомпилированный файл shelter.war из папки target/ вашего проекта.

Перенесите его в папку веб-приложений Tomcat: apache-tomcat-10.1.x/webapps/.

Переименуйте файл в ROOT.war, если хотите, чтобы приложение открывалось по корневому адресу / без указания префикса в URL.

4. Запуск сервера:

Windows: Запустите bin/catalina.bat run или bin/startup.bat

Linux / macOS: Выполните ./bin/catalina.sh run

После успешного старта Tomcat приложение будет доступно в браузере по адресу:
👉 http://localhost:8083/
