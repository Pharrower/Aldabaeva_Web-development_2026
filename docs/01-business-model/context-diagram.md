# Контекстная диаграмма системы (Context Diagram / IDEF0)

## Описание
Контекстная диаграмма определяет границы информационной системы приюта «Доброе сердце», идентифицируя внешние сущности (акторов), которые взаимодействуют с системой, а также потоки данных между ними. Моделирование выполнено в соответствии с методологией структурного анализа IDEF0.

## Визуализация диаграммы

## Потоки данных и сущности
1. **Потенциальный владелец (Гость сайта):**
   * *Входящий поток:* Просмотр каталога доступных животных, просмотр уровней заполненности склада.
   * *Исходящий поток:* Отправка асинхронной AJAX-заявки на усыновление животного (содержит ФИО, телефон, сообщение).

2. **Волонтер / Администратор:**
   * *Входящий поток:* Просмотр списка поступивших заявок, управление карточками животных, обновление данных о материальном обеспечении на складе.
   * *Исходящий поток:* Добавление/удаление записей, изменение статусов заявок (`Одобрено`, `Отклонено`).

3. **Управляющие воздействия и механизмы:**
   * *Управление (Сверху):* Нормативно-правовые акты, устав приюта, правила санитарного контроля.
   * *Механизмы (Снизу):* Волонтёрский персонал, аппаратное обеспечение (сервер), СУБД PostgreSQL.
