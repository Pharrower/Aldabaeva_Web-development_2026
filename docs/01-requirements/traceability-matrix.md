# Матрица трассировки требований (RTM)

Данная матрица связывает бизнес-цели и сценарии использования (Use Cases) с конкретными функциональными блоками системы. Это позволяет гарантировать, что каждое требование реализовано в коде.

| ID     | Наименование Use Case     | Компонент системы                         | Статус    |
| :----- | :------------------------ | :---------------------------------------- | :-------- |
| REQ-01 | Учет животных             | `AnimalService` / `RestApiController`     | Выполнено |
| REQ-02 | Управление складом        | `SupplyService` / `RestApiController`     | Выполнено |
| REQ-03 | Учет пожертвований        | `DonationService` / `RestApiController`   | Выполнено |
| REQ-04 | Волонтерская деятельность | `VolunteerService` / `RestApiController`  | Выполнено |
| REQ-05 | Статистика приюта         | `AnimalService` (method: getAdoptedCount) | Выполнено |

## Пояснения к матрице:
* **REQ-XX**: Уникальный идентификатор требования, соответствующий диаграмме Use Case.
* **Компонент системы**: Основной класс или сервис, отвечающий за логику данного требования.
* **Статус**: Текущий уровень готовности (Выполнено / В работе).
