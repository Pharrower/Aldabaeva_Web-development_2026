# Функциональная диаграмма прецедентов (Use Case Diagram)

## Описание
Диаграмма прецедентов визуализирует ожидаемое поведение информационной системы с точки зрения различных категорий пользователей (Акторов). Моделирование выполнено в нотации UML.

## Визуализация диаграммы
Ниже представлена функциональная схема системы:

![Диаграмма прецедентов UML](images/use-case-diagram.png)

## Код модели (PlantUML)

```plantuml
@startuml
skinparam packageStyle rectangle
skinparam actorStyle awesome
skinparam shadowing false
skinparam linetype ortho

' Корректное указание горизонтальной ориентации
left to right direction

actor "Пользователь" as User #E8F0FE
actor "Администратор" as Admin #FCE8E6

rectangle "Информационная система приюта ""Доброе сердце""" as System #F1F3F4 {
  (Просмотр каталога животных) as UC1
  (Подача заявки на адопцию) as UC2
  (Просмотр мониторинга нужд склада) as UC3
  (Оформление пожертвования) as UC4
  (Аутентификация / Вход) as UC5
  
  (Управление карточками животных) as UC6
  (Модерация заявок на усыновление) as UC7
  (Учет материальных запасов склада) as UC8
  (Просмотр финансовой аналитики) as UC9
}

' Связи для обычного пользователя
User --> UC1
User --> UC2
User --> UC3
User --> UC4
User --> UC5

' Связи для администратора
Admin --> UC5
Admin --> UC6
Admin --> UC7
Admin --> UC8
Admin --> UC9

' Отношения включения
UC2 ..> UC5 : <<include>>
UC7 ..> UC5 : <<include>>

@enduml
```

## Спецификация акторов
* **Пользователь (Гость / Потенциальный владелец):** Неавторизованный посетитель веб-интерфейса. Имеет доступ к чтению каталога животных, просмотру прогресс-баров склада и отправке асинхронных заявок.
* **Администратор (Волонтер / Сотрудник приюта):** Авторизованный пользователь с расширенными привилегиями. Осуществляет операционный учет фонда животных, управление заявками и инвентаризацию материальных ресурсов.
