# Спецификации прецедентов использования (Use Case Specifications)

## UC-1: Подача заявки на усыновление питомца (Асинхронный сценарий)
* **Актор:** Потенциальный владелец (Пользователь).
* **Предусловие:** Пользователь открыл карточку конкретного животного в каталоге.
* **Основной успешный сценарий:**
  1. Пользователь нажимает кнопку «Забрать домой», система открывает веб-форму.
  2. Пользователь вводит контактные данные и сообщение.
  3. Клиентский скрипт отправляет асинхронный AJAX-запрос на сервер.
  4. Сервер проверяет данные через Jakarta Bean Validation.
  5. Данные фиксируются в таблице `adoption_requests` со статусом `PENDING`.
  6. Интерфейс без перезагрузки отображает уведомление об успехе.
* **Альтернативные сценарии:**
  * *Исключение 3а (Короткое сообщение):* JavaScript блокирует отправку и выводит ошибку валидации.

## UC-2: Обновление данных склада снабжения
* **Актор:** Администратор.
* **Предусловие:** Администратор успешно авторизован и перешел в раздел «Склад».
* **Основной успешный сценарий:**
  1. Администратор изменяет значения полей `current_amount` или `target_amount`.
  2. Система отправляет POST-запрос в `ShelterAdminController`.
  3. Сервис `AnimalService` сохраняет состояние в БД.
  4. Пользователь перенаправляется на обновленную страницу склада.

## UC-3: Подача анкеты волонтера
* **Актор:** Пользователь.
* **Предусловие:** Пользователь находится на странице регистрации волонтеров.
* **Основной успешный сценарий:**
  1. Пользователь заполняет форму анкеты.
  2. Система сохраняет запись в таблице `volunteer_requests`.
  3. Статус заявки устанавливается как `PENDING`.
  4. Администратор получает уведомление о новой анкете.

## UC-4: Оформление денежного пожертвования
* **Актор:** Пользователь.
* **Предусловие:** Пользователь перешел в раздел финансовой помощи.
* **Основной успешный сценарий:**
  1. Пользователь указывает сумму пожертвования.
  2. Система фиксирует транзакцию в таблице `donations` с привязкой к текущему пользователю и текущей дате (`TIMESTAMP`).
  3. Формируется запись в истории пожертвований, доступная для просмотра в аналитике.
