# ADR-001: Выбор паттерна PCMEF для организации слоев приложения

## Статус
Принято (Accepted).

## Контекст
При проектировании архитектуры системы возникла необходимость обеспечить высокую поддерживаемость и тестируемость кода. Необходима структура, которая четко разделяет ответственность между обработкой запросов, бизнес-логикой и хранением данных, учитывая использование фреймворка Spring Boot.

## Решение
Выбран паттерн **PCMEF** (Presentation-Control-Mediator-Entity-Foundation):
* **Presentation:** Шаблоны Thymeleaf.
* **Control:** Spring MVC Controllers.
* **Mediator:** Сервисный слой (@Service).
* **Entity:** JPA Entities.
* **Foundation:** Spring Data Repositories.

## Последствия
### Преимущества:
- **Разделение ответственности:** Изменения в базе данных (Foundation) не требуют правок в пользовательском интерфейсе (Presentation).
- **Тестируемость:** Возможность тестирования бизнес-логики (Mediator) отдельно от веб-слоя.
- **Масштабируемость:** Четкая структура позволяет легко добавлять новые функциональные модули.

### Недостатки:
- **Бойлерплейт:** Требуется создание нескольких классов для реализации даже простых операций (CRUD).
