# ADR-002: Выбор базы данных (PostgreSQL) и ORM (Spring Data JPA)

## Статус
Принято (Accepted).

## Контекст
Для хранения данных системы приюта «Доброе сердце» (карточки животных, заявки, складские запасы) требуется надежная система управления базами данных. Необходимо минимизировать объем рутинного кода для выполнения CRUD-операций и обеспечить целостность данных.

## Решение
Выбрана следующая связка:
1. **СУБД:** PostgreSQL (реляционная база данных с поддержкой сложных запросов и транзакций).
2. **ORM:** Spring Data JPA (Hibernate в качестве провайдера).

## Последствия
### Преимущества:
- **Производительность:** PostgreSQL отлично справляется со сложными связями между таблицами (например, связь "Волонтер" — "Заявка" — "Животное").
- **Ускорение разработки:** Spring Data JPA позволяет генерировать запросы к БД на основе имен методов интерфейсов, что избавляет от написания большого количества шаблонного SQL-кода.
- **Безопасность:** JPA защищает от SQL-инъекций за счет использования параметризованных запросов.

### Недостатки:
- **Оверхед (Overhead):** ORM потребляет больше памяти, чем прямое использование JDBC, однако для текущих масштабов проекта это не является критическим фактором.
- **Скрытые запросы:** Необходимо следить за тем, какие запросы генерирует Hibernate (проблема N+1), чтобы избежать замедления системы.
