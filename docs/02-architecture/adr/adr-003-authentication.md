# ADR-003: Стратегия аутентификации и авторизации

## Статус
Принято (Accepted).

## Контекст
Системе приюта «Доброе сердце» требуется механизм защиты административного интерфейса и управления правами доступа (волонтеры vs администраторы). Необходимо обеспечить безопасность пользовательских данных, включая пароли, и предотвратить несанкционированный доступ к функциям управления заявками и складом.

## Решение
Использование модуля **Spring Security** в сочетании со стандартной сессионной моделью аутентификации:
* **Аутентификация:** Проверка учетных данных (логин/пароль) из базы данных PostgreSQL.
* **Хеширование:** Использование алгоритма `BCryptPasswordEncoder` для хранения паролей.
* **Авторизация:** Управление доступом на основе ролей (`ROLE_USER`, `ROLE_ADMIN`) через аннотации Spring Security (`@PreAuthorize` или конфигурацию `HttpSecurity`).

## Последствия
### Преимущества:
- **Надежность:** Spring Security является промышленным стандартом безопасности для Java-приложений.
- **Безопасность:** Защита от CSRF-атак включена «из коробки», что крайне важно для форм в Thymeleaf.
- **Простота интеграции:** Бесшовная работа с имеющимися JPA-сущностями пользователей.

### Недостатки:
- **Сложность настройки:** Требует детальной конфигурации цепочки фильтров безопасности, что может быть нетривиально для новичка.
