# Спецификации прецедентов использования (Use Case Specifications)

## UC-1: Подача заявки на усыновление питомца (Асинхронный сценарий)
* **Актор:** Потенциальный владелец (Пользователь).
* **Предусловие:** Пользователь открыл карточку конкретного животного в каталоге.
* **Основной успешный сценарий:**
  1. Пользователь нажимает кнопку «Забрать домой», система открывает веб-форму.
  2. Пользователь вводит ФИО, контактный телефон и сопроводительное сообщение.
  3. На стороне клиента JavaScript проверяет длину сообщения (не менее 10 символов).
  4. Клиентский скрипт отправляет асинхронный AJAX-запрос на сервер.
  5. Серверный контур Spring Boot повторно валидирует объект через Jakarta Bean Validation.
  6. Данные успешно фиксируются в таблице `adoption_requests` со статусом `PENDING`.
  7. Сервер возвращает HTTP-статус 200 (OK).
  8. Интерфейс страницы без перезагрузки скрывает форму и отображает зеленую плашку об успешной отправке.
* **Альтернативные сценарии (Обработка ошибок):**
  * *Исключение 3а (Короткое сообщение):* JavaScript блокирует отправку, подсвечивает текстовое поле красным (`is-invalid`) и выводит ошибку «Сообщение должно содержать не менее 10 символов».
  * *Исключение 5а (Обход клиентской защиты):* Если запрос отправлен в обход JS, сервер перехватывает ошибку валидации, отклоняет транзакцию и возвращает пользователю страницу с указанием ошибок формы.

## UC-2: Обновление данных склада снабжения
* **Актор:** Администратор (Волонтер).
* **Предусловие:** Волонтер успешно авторизован в системе и перешел в раздел «Склад».
* **Основной сценарий:**
  1. Волонтер указывает новое текущее количество ресурса (например, корма).
  2. Система пересчитывает процент заполненности склада методом `getPercent()`.
  3. Изменения сохраняются в БД.
  4. Главная страница динамически адаптирует шкалу Progress Bar под новые значения.
