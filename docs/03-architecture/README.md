# Этап 3: Проектирование архитектуры программной системы (Architectural Design)

## Описание этапа
На данном этапе осуществлено проектирование логической архитектуры веб-ориентированного приложения приюта «Доброе сердце». Согласно жестким требованиям методических указаний по дисциплине «Программная инженерия», в качестве архитектурного фундамента использован расширенный шаблон **PCMEF** (Presentation-Control-Mediator-Entity-Foundation). Такой подход обеспечивает слабую связанность компонентов (Loose Coupling), высокую модульность, независимость слоев и упрощает долгосрочное сопровождение программного продукта.

* [interfaces.md](interfaces.md) — Документирование межслойных интерфейсов, структуры пакетов и логики распределения обязанностей.

## Архитектурный стиль
Приложение спроектировано как модульный веб-монолит с использованием концепции **Server-Side Rendering (SSR)**. Контроллеры бэкенда перехватывают HTTP-запросы от пользователя, взаимодействуют со слоем бизнес-логики и возвращают полностью сформированные HTML-страницы (шаблоны Thymeleaf), обогащенные данными из базы данных PostgreSQL. Взаимодействие со слоем данных изолировано и оптимизировано пулом соединений HikariCP.
