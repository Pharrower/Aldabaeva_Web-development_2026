# Спецификация межслойного взаимодействия и интерфейсов

## Логическая структура пакетов (Package Structure)
Исходный код разработанного Spring Boot приложения структурирован по пакетам, соответствующим слоям PCMEF:
* `com.example.shelter.controllers` — классы управления потоками, перехватчики веб-форм.
* `com.example.shelter.services` — интерфейсы и реализации компонентов бизнес-логики.
* `com.example.shelter.entities` — классы-сущности (Data Model).
* `com.example.shelter.repositories` — интерфейсы доступа к данным, расширяющие `JpaRepository`.

## Шаблоны межслойного взаимодействия
Взаимодействие между слоями строго регламентировано и осуществляется сверху вниз с применением паттерна **Dependency Injection (Внедрение зависимостей)** средствами Spring Framework:

1. **Presentation ──> Control:** Браузер инициирует HTTP POST-запрос с данными формы или AJAX Fetch-запрос. Контроллер принимает данные.
2. **Control ──> Mediator (Service):** Контроллер валидирует входящие параметры и вызывает соответствующий метод сервиса (бизнес-логики), например, `supplyService.updateQuantity()`. Прямой доступ из контроллера в репозиторий запрещен.
3. **Mediator ──> Foundation (Repository):** Сервисный слой открывает транзакцию (аннотация `@Transactional`) и вызывает методы репозитория для персистентного сохранения объектов в СУБД PostgreSQL.

## Изоляция слоев и безопасность данниых
Благодаря жесткому разделению на интерфейсы и реализации в слое Foundation (Spring Data JPA генерирует прокси-классы автоматически), бизнес-логика полностью абстрагирована от специфики написания низкоуровневых SQL-запросов, что гарантирует защиту от атак типа SQL-Injection и обеспечивает переносимость системы на другие СУБД.
