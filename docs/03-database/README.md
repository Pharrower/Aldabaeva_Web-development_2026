# Проектирование и реализация базы данных (Database Design)

## Описание этапа
На данном этапе осуществлен переход от концептуальной доменной модели предметной области к физическому проектированию реляционной базы данных. В качестве целевой СУБД используется высокопроизводительная объектно-реляционная система **PostgreSQL 17**. Взаимодействие приложения со слоем СУБД абстрагировано средствами объектно-реляционного отображения (ORM Hibernate / Spring Data JPA) и оптимизировано за счет внедрения пула соединений **HikariCP**.

## Стратегия обеспечения целостности данных
1. **Первичные ключи (Primary Keys):** Для всех таблиц используются суррогатные автоинкрементные ключи типа `BIGSERIAL` (`bigint`), что гарантирует уникальность записей и высокую скорость индексации.
2. **Внешние ключи (Foreign Keys):** Целостность связей между таблицами обеспечивается явным объявлением ограничений `FOREIGN KEY` с каскадным удалением зависимых сущностей там, где это необходимо (`ON DELETE CASCADE`).
3. **Ограничения уникальности (Unique Constraints):** Поле `username` в таблице пользователей снабжено признаком `UNIQUE` для предотвращения дублирования аккаунтов.
