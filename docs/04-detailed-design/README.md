# Детальное проектирование динамики системы (Detailed Design)

## Описание этапа
Если на этапе архитектурного анализа была определена статическая структура слоев PCMEF, то в рамках детального проектирования (Detailed Design) формализуется поведенческая (динамическая) модель информационной системы приюта «Доброе сердце». Основная цель — детально описать временную последовательность передачи сообщений, вызовов методов и обработки исключений между компонентами при выполнении ключевых пользовательских сценариев.

## Содержимое каталога
* [sequence-diagrams.md](sequence-diagrams.md) — Спецификация диаграмм последовательности (UML Sequence Diagrams) с пошаговым описанием жизненного цикла запросов.
* [api-specification.md](api-specification.md) — Техническое описание эндпоинтов, форматов передаваемых данных (DTO) и контрактов взаимодействия.

## Ключевые аспекты проектирования динамики
1. **Синхронность и асинхронность:** Описана гибридная модель, где базовая маршрутизация страниц происходит в синхронном режиме Spring MVC, а взаимодействие с критически важными формами (заявки на адопцию) изолировано для улучшения пользовательского опыта (AJAX / Fetch API).
2. **Транзакционность:** Слой Mediator (Services) выступает в роли разграничителя транзакций (`@Transactional`). Запись в базу данных PostgreSQL фиксируется только при успешном прохождении бизнес-логики и межслойных проверок.
3. **Разделение контрактов (DTO):** Для защиты доменной модели и минимизации лишних данных, взаимодействие между слоями (Control ↔ Mediator) осуществляется исключительно через DTO (Data Transfer Objects), что обеспечивает гибкость системы при изменении структуры БД.
4. **Централизация обработки исключений:** Вся логика обработки ошибок (отсутствие записи, ошибки валидации, ошибки подключения к БД) вынесена в специализированные обработчики, предотвращая «размазывание» блоков `try-catch` по всей бизнес-логике.
