# Спецификация API (Controller Endpoints)

В данном разделе приведено описание эндпоинтов, предоставляемых контроллерами системы для обработки запросов.

## 1. Контроллер управления питомцами (AnimalController)

| Метод | Путь | Описание | Параметры / Тело |
| :--- | :--- | :--- | :--- |
| `GET` | `/animals` | Получение списка всех питомцев | - |
| `GET` | `/animals/{id}` | Детальная страница питомца | `id` (Path variable) |
| `POST` | `/animals/add` | Регистрация нового питомца | `AnimalDTO` (Request body) |

## 2. Контроллер заявок на адопцию (AdoptionController)

| Метод | Путь | Описание | Параметры / Тело |
| :--- | :--- | :--- | :--- |
| `POST` | `/adoption/submit` | Подача заявки на усыновление | `AdoptionDTO` (Form data) |
| `GET` | `/adoption/list` | Просмотр всех заявок (для ADMIN) | - |

## 3. Контроллер складского учета (SupplyController)

| Метод | Путь | Описание | Параметры / Тело |
| :--- | :--- | :--- | :--- |
| `GET` | `/supplies` | Список доступных ресурсов | - |
| `POST` | `/supplies/update` | Обновление количества ресурса | `Long id`, `Integer amount` |

---

## Формат данных (Data Transfer Objects)
Для взаимодействия между слоями используются DTO-объекты, которые исключают прямое использование JPA-сущностей в контроллерах:

* **AdoptionDTO:**
  * `animalId` (Long): ID питомца.
  * `applicantName` (String): Имя заявителя.
  * `applicantPhone` (String): Контактный телефон.
  * `message` (String): Дополнительная информация.

* **SupplyDTO:**
  * `id` (Long): ID товара.
  * `currentQuantity` (Integer): Новое количество.

---
*Спецификация отражает контракт между слоем представления (Presentation) и слоем управления (Control), обеспечивая четкую типизацию передаваемых данных.*
