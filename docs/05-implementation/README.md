# Реализация (Implementation)

## Описание
В данном разделе представлена структура исходного кода системы, принципы организации пакетов и механизмы межслойного взаимодействия, реализованные в соответствии с паттерном PCMEF.

## Содержимое каталога
* [package-structure.md](package-structure.md) — Описание иерархии пакетов и принципов именования компонентов.
* [class-diagram.md](class-diagram.md) — Диаграмма классов проектирования, демонстрирующая связи между компонентами системы.
* [layer-interaction-example.md](layer-interaction-example.md) — Демонстрация сквозного вызова от контроллера к БД через сервисный слой.

## Технологический стек
- **Язык:** Java 17.
- **Фреймворк:** Spring Boot 3.x.
- **Инструментарий:** 
    - **Lombok** — для генерации boilerplate-кода (геттеры, сеттеры, конструкторы).
    - **Spring Data JPA** — для автоматизации работы с PostgreSQL.
    - **Thymeleaf** — серверный движок шаблонов.

## Принципы реализации
1. **Dependency Injection (DI):** Внедрение зависимостей реализовано через конструкторы (`@RequiredArgsConstructor`), что обеспечивает неизменяемость компонентов (`final` поля) и упрощает юнит-тестирование.
2. **Слабая связанность:** Компоненты взаимодействуют исключительно через интерфейсы (например, репозитории), что позволяет легко модифицировать инфраструктурный слой без правок в бизнес-логике.
3. **Разделение слоев (PCMEF):** Реализована строгая изоляция: контроллеры не содержат бизнес-логики, а сервисы не зависят от деталей HTTP-запросов.
