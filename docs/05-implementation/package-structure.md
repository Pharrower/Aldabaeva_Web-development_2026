# Структура реализации (Source Code Structure)

Проект организован по пакетной структуре, соответствующей архитектурному паттерну PCMEF. Все классы расположены в пакете `com.example.shelter`.

## Организация пакетов
* `config/` — конфигурация безопасности (Spring Security) и инфраструктуры.
* `controller/` — Spring MVC контроллеры (Presentation Layer).
* `service/` — бизнес-логика и интерфейсы сервисов (Mediator Layer).
* `repository/` — Spring Data интерфейсы (Foundation Layer).
* `model/` — JPA сущности (Entity Layer).
* `dto/` — объекты передачи данных между слоями.

## Правила взаимодействия
1. **Dependency Injection:** Все зависимости внедряются через конструкторы (`@RequiredArgsConstructor`), что обеспечивает неизменяемость полей и облегчает тестирование.
2. **Инкапсуляция:** Контроллеры работают только с сервисами, сервисы — только с репозиториями.
