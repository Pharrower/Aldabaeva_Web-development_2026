# Отчет об автоматизированном модульном тестировании (Unit Testing)

## Описание
Для верификации корректности работы разработанного программного обеспечения, исключения регрессионных ошибок при модернизации кода и подтверждения надежности вычислений было проведено модульное тестирование ключевых методов бизнес-логики. Тестирование реализовано с применением библиотеки **JUnit 5 (Jupiter)**.

## Описание тестовых сценариев

### Тест 1: Верификация логики расчета заполненности склада (`SupplyEntityTest`)
* **Цель:** Проверить правильность вычисления процента заполнения ячейки склада методом `getPercent()`.
* **Входные данные:** Текущее количество `currentQuantity = 75`, максимальная емкость `maxCapacity = 100`.
* **Ожидаемый результат:** Метод должен вернуть целочисленное значение `75`.
* **Исходный код теста:**
```java
@Test
public void testGetPercentCalculation() {
    Supply supply = new Supply();
    supply.setCurrentQuantity(75);
    supply.setMaxCapacity(100);
    
    int percent = supply.getPercent();
    
    Assertions.assertEquals(75, percent, "Математический расчет процента заполнения некорректен!");
}
```

### Тест 2: Проверка обработки граничных условий (Пустая емкость)
* **Цель:** Исключить риск возникновения критической арифметической ошибки деления на ноль (ArithmeticException: / by zero) в случае, если максимальная емкость склада не была инициализирована.
* **Входные данные:** currentQuantity = 0, maxCapacity = 0.
* **Ожидаемый результат:** Безопасная обработка граничного условия (метод возвращает 0).
