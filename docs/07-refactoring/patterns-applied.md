# Примененные паттерны проектирования

1. **Service Layer Pattern:** Мы создали четкую границу между тем, как данные отображаются (Controller) и как они обрабатываются (Service).
2. **Dependency Injection (Constructor Injection):** Применен для обеспечения неизменяемости зависимостей и упрощения юнит-тестирования.
3. **Facade Pattern:** Сервисы теперь выступают как «фасады» для своих репозиториев, скрывая сложность запросов к БД от контроллеров.
