# План рефакторинга проекта "Shelter"

## 1. Проблематика (Code Smells)
- **God Object:** `AnimalService` был перегружен логикой, не относящейся к животным (донаты, поставки, волонтеры).
- **Нарушение SRP (Single Responsibility Principle):** Контроллеры напрямую обращались к репозиториям, минуя сервисный слой.
- **Fragile Dependencies:** Использование `@Autowired` над полями делало код зависимым от Spring-контекста и усложняло тестирование.

## 2. Основные этапы улучшения
1. **Декомпозиция:** Разделение монолитного сервиса на `AnimalService`, `DonationService`, `VolunteerService`, `SupplyService` и `UserService`.
2. **Инверсия управления:** Переход к конструкторной инъекции зависимостей для всех компонентов.
3. **Изоляция слоев:** Перенос бизнес-логики (подсчет сумм, фильтрация) из контроллеров в сервисы.
