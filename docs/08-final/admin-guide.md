# Руководство системного администратора по развертыванию ИС

## 1. Состав программного обеспечения сервера
Для промышленного развертывания веб-приложения на целевом сервере должны быть предварительно установлены:
1. СУБД **PostgreSQL 17** (порт по умолчанию `5432`).
2. Среда выполнения **Java SDK 25**.
3. Сервер приложений (контейнер сервлетов) **Apache Tomcat 10.1.x**.

## 2. Развертывание базы данных
1. Создайте в СУБД пустую базу данных:
   ```sql
   CREATE DATABASE shelter_db;
    ```
2. Импортируйте структуру таблиц, ограничений и индексов, выполнив скрипт из корня репозитория:
```Bash
psql -U postgres -d shelter_db -f docs/04-database/ddl.sql
```
3. Компиляция и сборка приложения
Сборка выполняется на машине разработчика или сервере непрерывной интеграции (CI) при помощи Maven. В корне проекта выполните команду:
```Bash
mvn clean package
```
В результате в папке target/ будет сгенерирован исполняемый архив shelter.war.
4. Конфигурация и запуск сервера Apache Tomcat 10.1
Настройка сетевого порта: Во избежание конфликтов портов в операционной системе, откройте файл конфигурации Tomcat conf/server.xml и переведите HTTP-коннектор на выделенный сетевой порт 8082:
```XML
<Connector connectionTimeout="20000" maxParameterCount="1000" port="8082" protocol="HTTP/1.1" redirectPort="8443" />
```
Деплой (Размещение артефакта):
Скопируйте скомпилированный файл shelter.war и поместите его в системный каталог веб-приложений Tomcat: apache-tomcat-10.1.x/webapps/. Чтобы приложение открывалось как корневое по адресу /, переименуйте файл в ROOT.war.

Запуск:

Для ОС Windows: запустите скрипт bin/startup.bat.

Для ОС Linux/macOS: выполните ./bin/catalina.sh run.

После старта веб-сервера информационная система будет доступна по сетевому адресу: http://localhost:8082/
