# Спецификация и методы документирования API

Для обеспечения высокого качества взаимодействия между клиентской и серверной частями приложения, в проекте была внедрена концепция **API-First** (документирование как основа разработки).

## 1. Спецификация OpenAPI (Swagger)
Мы используем библиотеку **Springdoc OpenAPI**, которая автоматически анализирует контроллеры Spring Boot и генерирует актуальную документацию в реальном времени.

### Основные преимущества:
* **Актуальность:** Документация всегда совпадает с кодом. Если метод в контроллере меняется, Swagger обновляется автоматически при пересборке проекта.
* **Интерактивность:** Разработчики фронтенда могут отправлять HTTP-запросы прямо из браузера через Swagger UI, не используя сторонние инструменты (типа Postman или cURL).

## 2. Аннотирование кода
Каждый эндпоинт в `RestApiController` сопровожден набором аннотаций, которые превращают сухой код в понятную документацию:

| Аннотация      | Описание                                                           |
| :------------- | :----------------------------------------------------------------- |
| `@Tag`         | Группирует эндпоинты в логические разделы (например, "REST API").  |
| `@Operation`   | Дает краткое резюме и описание назначения конкретного метода.      |
| `@ApiResponse` | Описывает возможные ответы сервера (статусы 200, 400, 404 и т.д.). |

## 3. Рекомендации по использованию API
1. **Авторизация:** В текущей версии API взаимодействие доступно для чтения. Для защищенных методов (если будут добавлены) необходимо передавать JWT-токен в заголовке `Authorization`.
2. **Формат данных:** Обмен данными осуществляется исключительно в формате `JSON`.
3. **Обработка ошибок:** В случае возникновения ошибки, сервер возвращает стандартные HTTP-коды, что позволяет клиенту корректно обрабатывать исключительные ситуации (например, показывать пользователю уведомление).
