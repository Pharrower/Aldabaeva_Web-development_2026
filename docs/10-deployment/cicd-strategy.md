# CI/CD Стратегия (GitHub Actions)

Автоматизация процесса сборки позволяет избежать человеческих ошибок при деплое.

## Процесс сборки (.github/workflows/deploy.yml)
1. **Checkout:** Получение кода из репозитория.
2. **Build:** Сборка проекта через Maven (`mvn clean package`).
3. **Test:** Прогон Unit-тестов (если они есть).
4. **Artifact:** Упаковка в WAR-файл.

*Теперь при каждом push в ветку main проект автоматически готов к отправке на сервер.*
